# 🔐 Authentication vs Authorization

## 📌 What is Authentication?
Authentication is the process of verifying the identity of a user.

👉 It answers:
**“Who are you?”**

---

## 📌 What is Authorization?
Authorization is the process of determining what an authenticated user is allowed to access.

👉 It answers:
**“What can you access?”**

---

## ⚙️ How it Works
1. User enters credentials (username/password)  
2. System verifies identity → Authentication  
3. System checks permissions → Authorization  
4. Access is granted or denied  

---

## 🔄 Key Differences

| Authentication | Authorization |
|---------------|--------------|
| Verifies identity | Grants access |
| First step | Second step |
| Username/password | Permissions/roles |
| Who you are | What you can do |

---

## 💥 Importance
- Protects systems from unauthorized access  
- Ensures users only access allowed resources  
- Core part of system security  

---

## 🛡️ Best Practices
- Use strong passwords  
- Enable **Multi-Factor Authentication (MFA)**  
- Apply **least privilege principle**  
- Use role-based access control (RBAC)  

---

## 🔗 Integration with IT Systems
- Active Directory manages authentication  
- GPO controls authorization  
- MFA adds additional security layer  

---

## 💼 Real-world Relevance
Authentication and authorization are used in all systems including Windows, cloud platforms, and applications.

---

## 🔥 Important Line
Authentication verifies identity, while authorization controls access.

---

## 🧠 Simple Understanding
Authentication = “Who are you?”  
Authorization = “What can you do?”

---

## 🎤 Interview Line
Authentication verifies a user’s identity, while authorization determines what resources that user is allowed to access.
