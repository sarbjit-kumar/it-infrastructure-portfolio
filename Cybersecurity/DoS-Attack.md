# 🌐 DoS (Denial-of-Service) Attack

## 📌 What is a DoS Attack?
A Denial-of-Service (DoS) attack is a cyber attack where an attacker floods a system, server, or network with excessive traffic, making it unavailable to legitimate users.

---

## ⚙️ How it Works
1. Attacker sends a large number of fake requests  
2. Server resources (CPU, memory, bandwidth) get overloaded  
3. System becomes slow or crashes  
4. Legitimate users cannot access the service  

---

## 💥 Impact / Damage
- Website or service downtime  
- Loss of revenue  
- Reduced system performance  
- Business disruption  

---

## 🛡️ Prevention / Protection
- Use firewalls and intrusion detection systems (IDS/IPS)  
- Implement rate limiting  
- Use load balancers  
- Monitor network traffic  
- Use DDoS protection services  

---

## 🧠 Key Concepts
- DoS = attack from a single source  
- Overloads system resources  
- Affects system availability  

---

## 🔄 DoS vs DDoS
- DoS → Single attacker  
- DDoS → Multiple systems (botnet)  

---

## 🔗 Integration with IT Systems
- Monitoring tools detect unusual traffic  
- Firewalls block malicious requests  
- Network policies improve security  

---

## 💼 Real-world Relevance
DoS attacks are commonly used to disrupt websites, online services, and business operations.

---

## 🔥 Important Line
A DoS attack overwhelms a system with traffic, making it unavailable to legitimate users.

---

## 🧠 Simple Understanding
DoS = “too much traffic = system down”

---

## 🎤 Interview Line
A DoS attack floods a system with excessive traffic, causing service disruption and preventing legitimate access.
