# 🔐 Multi-Factor Authentication (MFA)

## 📌 What is MFA?
Multi-Factor Authentication (MFA) is a security method that requires users to verify their identity using two or more authentication factors.

---

## 🎯 Purpose
- Add an extra layer of security  
- Protect accounts from unauthorized access  
- Reduce risk of stolen credentials  

---

## ⚙️ How it Works
1. User enters username and password  
2. System requests a second verification factor  
3. User provides additional verification (OTP, app approval, biometric)  
4. Access is granted if both factors are correct  

---

## 🔹 Types of Factors
- **Something you know** → Password  
- **Something you have** → Phone / OTP / Authenticator app  
- **Something you are** → Fingerprint / Face recognition  

---

## 💥 Importance
- Protects even if password is compromised  
- Reduces impact of phishing attacks  
- Widely used in enterprise and cloud systems  

---

## 🛡️ Best Practices
- Enable MFA on all important accounts  
- Use authenticator apps instead of SMS where possible  
- Avoid sharing verification codes  
- Regularly review login activity  

---

## 🔗 Integration with IT Systems
- Works with Authentication systems  
- Used in Active Directory and cloud platforms (Azure, AWS)  
- Enhances overall security policies  

---

## 💼 Real-world Relevance
MFA is widely used in organisations to secure user accounts, especially for email, banking, and cloud services.

---

## 🔥 Important Line
MFA adds an extra layer of security by requiring multiple forms of verification.

---

## 🧠 Simple Understanding
MFA = “extra security step after password”

---

## 🎤 Interview Line
Multi-Factor Authentication enhances security by requiring multiple verification methods before granting access.
