# 🖧 Network Design using Cisco Packet Tracer

## 📌 Overview

This section documents the practical network design created in Cisco Packet Tracer. The purpose of this task was to apply networking concepts such as IP addressing, subnetting, device configuration, and connectivity testing in a simulated environment.

Packet Tracer provides a practical way to design and test networks without needing physical hardware.

---

## 📌 Objective

The objective of this network design was to:

- Build a small network using routers, switches, and PCs  
- Apply subnetting to assign IP addresses correctly  
- Configure devices with IP address, subnet mask, and default gateway  
- Test connectivity between devices  

---

## 📌 Devices Used

The following devices were used in the Packet Tracer design:

- Router  
- Switch  
- PCs / End Devices  

---

## 📌 Network Configuration

Each device was configured with:

- IP address  
- Subnet mask  
- Default gateway  

Subnetting was used to divide the network into smaller segments and assign addresses efficiently.

---

## 📌 Connectivity Testing

After configuration, connectivity was tested using:

- `ping`

These tests confirmed that devices were able to communicate correctly across the network.

---

## 📌 Practical Outcome

The network was successfully designed and configured in Cisco Packet Tracer.

Results:

- Devices received correct IP configuration  
- Subnetting was applied correctly  
- Connectivity between devices was successful  
- Network communication was verified through ping tests
